# Настройка Keycloak для IrTRAN

Keycloak поднимается через Docker (см. [DOCKER.md](./DOCKER.md)). В этом документе — только настройка реалма и клиента для входа в приложение.

---

## Что уже задано через Docker и .env

- **Keycloak:** http://localhost:8080  
- **Логин админки:** из `.env` — по умолчанию `KEYCLOAK_ADMIN=admin`, `KEYCLOAK_ADMIN_PASSWORD=admin`
- **Реалм приложения:** в `.env` указан `KEYCLOAK_REALM=master` (можно сменить на свой реалм)
- **Клиент приложения:** в `.env` указан `KEYCLOAK_RESOURCE=irtran-client` — этот клиент нужно создать в выбранном реалме

Приложение (фронт и бэкенд) используют именно этот реалм и этот клиент. Если создадите реалм или клиент с другими именами — поменяйте соответствующие переменные в `.env` и перезапустите контейнеры.

---

## 1. Убедиться, что Keycloak запущен

```bash
docker-compose ps
```

Сервис `keycloak` должен быть в статусе `Up`. Если нет — `docker-compose up -d` из каталога `projirgups`.

Подождите 30–60 секунд после старта, затем откройте в браузере:  
**http://localhost:8080**

Должна открыться стартовая страница Keycloak (или сразу форма входа в админ-консоль).

---

## 2. Войти в админ-консоль

1. Нажмите **Administration Console** (или перейдите по ссылке входа в админку).
2. Введите логин и пароль из `.env`:
   - по умолчанию: **Username** `admin`, **Password** `admin`.

---

## 3. Выбрать реалм

В левом верхнем углу отображается текущий реалм (выпадающий список).

- Если в `.env` указано `KEYCLOAK_REALM=master` — оставьте реалм **master**.
- Если используете свой реалм (например, `irtran`) — выберите его в списке. При необходимости создайте реалм: **Создать область** → введите имя (например, `irtran`) → **Создать**. После создания не забудьте указать это имя в `.env` как `KEYCLOAK_REALM=irtran` и перезапустить backend/frontend при необходимости.

Все дальнейшие шаги выполняйте в том реалме, который указан в `KEYCLOAK_REALM`.

---

## 4. Создать клиента для приложения (обязательно)

Без этого шага вход в IrTRAN с фронта работать не будет.

1. В левом меню выберите **Клиенты** → **Создать клиента**.

2. **Вкладка «Общие настройки»:**
   - **Тип клиента:** оставьте **OpenID Connect**.
   - **Идентификатор клиента:** `irtran-client` (должен совпадать с `KEYCLOAK_RESOURCE` в `.env`).
   - Остальное можно не менять. Нажмите **Далее**.

3. **Вкладка «Возможности» (Capability config):**
   - **Аутентификация клиента:** **Выкл** (публичный клиент).
   - **Стандартный поток:** **Вкл**.
   - **Прякой доступ к выдаче разрешений:** **Вкл** (для входа через логин/пароль).
   - Остальные опции можно оставить по умолчанию. Нажмите **Далее**.

4. **Вкладка «Настройки входа» (Login settings):**
   - **Корневой URL:** можно оставить пустым или указать `http://localhost:5173`.
   - **Допустимые URI перенаправления:** добавьте по одному на строку:
     - `http://localhost:5173`
     - `http://localhost:5173/*`
     - `http://localhost:5173/silent-check-sso.html`
   - **Допустимые URI перенаправления после выхода:** например:
     - `http://localhost:5173`
     - `http://localhost:5173/*`
   - **Веб-источники:** `http://localhost:5173`
   Нажмите **Сохранить**.

> Зачем нужен `silent-check-sso.html`  
> Приложение использует “тихую проверку сессии” (silent SSO check), чтобы при обновлении страницы (F5) не попадать в цикл редиректов `check-sso`. Файл лежит в `IrTRAN-main/public/silent-check-sso.html`.

После сохранения клиент `irtran-client` появится в списке клиентов. Этого достаточно, чтобы проверить вход в приложение по адресу http://localhost:5173 (кнопка «Войти» → редирект на Keycloak → после входа возврат в приложение).

---

## 5. Создать пользователя для входа (если ещё нет)

По умолчанию в реалме `master` может не быть пользователей, кроме админа.

1. В левом меню: **Пользователи** → **Добавить пользователя**.
2. Заполните **Имя пользователя** (обязательно), при необходимости **Email**, **Имя**, **Фамилия**.
3. Нажмите **Создать**.
4. Откройте вкладку **Учётные данные** → задайте **Пароль** и при необходимости включите **Временный** (пользователь сменит при первом входе). Нажмите **Сохранить**.

Этого пользователя можно использовать для входа в IrTRAN через Keycloak.

---

## 6. Опционально: электронная почта (SMTP)

Нужно только если используете восстановление пароля или подтверждение email.

1. **Настройки области** (Realm settings) → вкладка **Электронная почта**.
2. Заполните хост, порт, от кого, включите аутентификацию при необходимости.
3. Для Gmail: включите 2FA и используйте [пароль приложения](https://myaccount.google.com/apppasswords), а не обычный пароль.

---

## 7. Опционально: подтверждение электронной почты

1. **Настройки области** → вкладка **Вход**: при необходимости включите **Регистрация пользователей**.
2. Вкладка **Электронная почта**: включите **Подтверждение электронной почты**.
3. **Аутентификация** → **Потоки** → **Регистрация**: убедитесь, что в потоке есть шаг проверки email.

---

## Краткая проверка

| Проверка | Действие |
|----------|----------|
| Keycloak доступен | Открыть http://localhost:8080 |
| Реалм совпадает с .env | В левом верхнем углу выбран реалм из `KEYCLOAK_REALM` |
| Клиент создан | В **Клиенты** есть `irtran-client` с Redirect URI `http://localhost:5173/*` |
| Есть пользователь | В **Пользователи** есть хотя бы один пользователь с паролем |
| Вход в приложение | Открыть http://localhost:5173 → «Войти» → авторизация в Keycloak → возврат в приложение |

При ошибках (например, «Invalid redirect URI») см. [KEYCLOAK_TROUBLESHOOTING.md](./KEYCLOAK_TROUBLESHOOTING.md).

Импорт и экспорт реалма описан в [KEYCLOAK_REALM_IMPORT_EXPORT.md](./KEYCLOAK_REALM_IMPORT_EXPORT.md).

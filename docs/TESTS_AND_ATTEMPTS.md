# Тесты, попытки и лимиты

Документ объясняет, как устроены тесты, конструктор, банк вопросов и **логика попыток** (включая лимиты).

## Сущности

- **Вопрос** (`questions`)
  - текст, тип (`radiogroup`/`checkbox`/`text`), варианты, правильный ответ
  - может иметь картинку (`image_path`)
- **Тест** (`tests`)
  - параметры: `pass_percent`, `max_attempts`, режим вариантов
- **Попытка** (`test_attempts`)
  - фиксирует факт прохождения и результат (процент/баллы)

## Режимы тестов

Поддерживаются два режима:

1. **Один вариант (случайный порядок)** — `single_shuffled`
2. **Несколько вариантов** — `per_variant`

## Лимит попыток: бизнес‑правила

В проекте действует правило:

- **попытка списывается при открытии теста** (даже если студент не отправил результат);
- “незавершённый тест” сам по себе **не считается попыткой**, но у нас нет “черновика” — попытка уже списана на старте;
- если тест “сдан”, **повторные попытки разрешены** (ограничение только по количеству);
- если преподаватель увеличит `max_attempts`, студент автоматически получит новые попытки (сравниваем текущий лимит с уже списанными попытками).

## Backend: API, связанный с попытками

### Запуск теста

- `GET /api/tests/:id/run`
  - отдаёт схему SurveyJS + `userAttemptCount` (сколько попыток уже списано)

### Списать попытку при старте

- `POST /api/tests/:id/attempts/start`
  - атомарно проверяет `max_attempts` и создаёт запись попытки со статусом `started`
  - если лимит исчерпан → `409 max_attempts_reached`

### Завершить попытку

- `PATCH /api/tests/:id/attempts/:attemptId/finish`
  - записывает результат в ранее созданную попытку, выставляет `finished_at`

## Frontend: что видит студент

В “Режиме теста” студент видит на карточках:

- **Осталось попыток** (если лимит задан)
- или **Безлимит**

Если попытки закончились:

- кнопка теста становится неактивной;
- показывается понятное сообщение.

## Frontend: инструменты преподавателя

### Банк заданий

- поиск по тексту/тегам
- поиск по **ID**
- автопоиск по ID (debounce)

### Конструктор тестов

- создание тестов
- поиск вопросов по **ID** в банке
- список существующих тестов:
  - поиск по **ID**
  - редактирование теста
  - удаление теста (с подтверждением)

Важно: удаление теста удаляет и попытки студентов по этому тесту.

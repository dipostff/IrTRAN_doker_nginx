# Заполнение справочников: импорт из файла и шаблоны

Этот документ описывает функцию **«Загрузить из файла»** в модуле “Заполнение справочников”.

## Зачем это нужно

Ручной ввод больших справочников занимает часы. Импорт позволяет:

- быстро залить данные из Excel/JSON;
- обновлять существующие записи по ключевым полям;
- получать отчёт по результатам (сколько добавлено/обновлено/пропущено).

## Форматы файлов

Поддерживаются:

- **JSON** (`.json`)
- **Excel** (`.xlsx`)

### JSON

- В JSON **флаги (названия полей) должны быть на английском**.
- Можно загружать:
  - массив объектов;
  - объект с ключом `items`;
  - специальный формат наподобие `nodes_map.json` для станций (где данные лежат в отдельных верхнеуровневых ключах).

### XLSX

- Заголовки столбцов могут быть **на русском или английском**.
- Импорт сначала нормализует заголовки, потом сопоставляет их с полями БД.

## Как работает сопоставление полей

Сервер строит mapping “заголовок в файле → поле таблицы”:

- **JSON**: строгий режим (ожидаются английские ключи).
- **XLSX**: допускается RU/EN; русские заголовки приводятся к ожидаемым.

Если в файле нет обязательных полей, строка будет пропущена (в отчёте будет видно почему).

## Кнопки на фронте

В секции “Обновление справочников” доступны:

- **Загрузить из файла** — выбрать `.json` или `.xlsx` и отправить на сервер
- **Скачать шаблон JSON**
- **Скачать шаблон XLSX**

Шаблоны генерируются сервером под каждый справочник: это самый безопасный способ подготовить файл.

## Backend: API

- `POST /api/dictionaries/:key/import`
  - принимает файл и выполняет upsert
  - возвращает статистику: total/inserted/updated/skipped + ошибки по строкам

- `GET /api/dictionaries/:key/template`
  - отдаёт шаблон (JSON или XLSX — зависит от параметров/кнопки)

## Практические советы

- Начните с “Скачать шаблон” и заполните его — это снижает шанс ошибок.
- Для числовых полей пустая строка автоматически приводится к `NULL`, чтобы не ловить ошибки вида “Incorrect integer value: ''”.
- Если строка слишком длинная для `VARCHAR`, она будет пропущена и попадёт в отчёт как `skipped` (импорт не упадёт целиком).

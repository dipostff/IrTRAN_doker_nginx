# Группы студентов и успеваемость (Keycloak Groups)

Этот документ описывает реализацию **групп студентов** через **Keycloak Groups** и то, как преподаватель смотрит **успеваемость группы** по тестам и сценариям.

## Зачем так сделано

Мы выбрали вариант “группы = Keycloak groups”, потому что:

- группы живут в SSO и могут использоваться не только в IrTRAN;
- membership не нужно хранить в MySQL;
- преподаватели могут владеть “своими” группами независимо друг от друга.

## Роли и доступ

- **teacher**: создаёт/удаляет группы, добавляет/удаляет студентов, смотрит успеваемость по своим группам.
- **app-admin**: может управлять группами всех преподавателей (служебный доступ).

## Как устроено владение группами

Каждый преподаватель имеет свои группы в “ветке” Keycloak. Логика владения определяется **путём группы** (group path).

Рекомендуемая структура:

- `IrTRAN/Teachers/<teacherUserId>/Groups/<groupName>`

Это даёт простой контроль доступа:

- преподаватель видит только группы внутри своего `teacherUserId`;
- администратор может видеть/управлять всеми.

## Backend: основные API

Все эндпоинты требуют токен Keycloak (`Authorization: Bearer ...`) и роль `teacher` или `app-admin`.

### Управление группами

- `GET /api/teacher/groups`
  - преподаватель: вернёт только его группы
  - админ: можно запросить все группы через `?all=1`
- `POST /api/teacher/groups`
  - тело: `{ "name": "Группа 1" }`
- `DELETE /api/teacher/groups/:groupId`

### Участники группы

- `GET /api/teacher/groups/:groupId/members`
- `POST /api/teacher/groups/:groupId/members`
  - тело: `{ "studentIds": ["<keycloakUserId>", "..."] }`
- `DELETE /api/teacher/groups/:groupId/members/:studentId`

### Успеваемость группы

- `GET /api/teacher/groups/:groupId/performance`

Возвращает:

- список участников;
- список тестов с агрегатами по группе + статусом по каждому студенту (`passed/failed/not_attempted`);
- список сценариев с агрегатами + флагом `viewed/notViewed` по каждому студенту.

## Frontend: Панель преподавателя

Во вкладке “Группы” преподаватель может:

- выбрать группу;
- создать группу;
- удалить группу;
- управлять участниками (добавить/удалить студента);
- смотреть успеваемость:
  - по тестам (с фильтрами и быстрыми счётчиками);
  - по сценариям (viewed/not viewed, “Проблемные”).

### “Профиль” студента

В таблицах успеваемости доступна кнопка **«Профиль»** — она открывает модальное окно с деталями по выбранному студенту.

### Прогресс тренажёра (учебные сценарии документов)

Студент в режиме «Новичок»/«Продвинутый» при заполнении заявки, накладной и др. отправляет на сервер снимок прогресса шагов сценария (с задержкой ~2 с). В таблице **«Тренажёр документов»** для выбранной группы показывается последний процент выполнения по каждому типу документа на студента.

- `POST /api/training/scenario-progress` — любой авторизованный пользователь (тело: `docType`, `docRef`, `doneCount`, `totalCount`, `doneStepIds`).
- `GET /api/teacher/groups/:groupId/training-scenario-progress` — роль `teacher` или `app-admin`, проверка владения группой.

Таблица БД: `training_scenario_snapshots` (миграция `20260328120000-TrainingScenarioSnapshots.js`). После обновления кода выполните миграции Sequelize в каталоге backend.

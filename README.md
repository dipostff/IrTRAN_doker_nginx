## Общая информация о проекте

Проект `projirgups` — это **ОТРЭД** (онлайн-тренажёр по работе с электронной документацией; в репозитории также используется техническое имя IrTRAN) по работе с железнодорожной транспортной документацией.  
Система имитирует полный цикл оформления, хранения и анализа документов, а также включает режим тестирования, справочники и вспомогательные сервисные модули.

Архитектура:
- **Фронтенд**: Vue 3 + Vite, Bootstrap / кастомные компоненты.
- **Бэкенд**: Node.js + Express, Sequelize (MySQL).
- **Аутентификация и роли**: Keycloak (realm‑роли `student`, `teacher`, `app-admin`, `dictionary-admin`).
- **Тесты**: SurveyJS (survey-core, survey-vue3-ui).

## Роли и доступ

- **student**
  - Доступ к основным учебным модулям по документам.
  - Прохождение тестов, просмотр результатов и журнала неправильных ответов.
  - Просмотр загруженных/сохранённых документов.
- **teacher**
  - Всё, что доступно студенту.
  - Панель преподавателя: просмотр студентов, их попыток и результатов тестов, состояние прохождения кейсов/сценариев.
  - Конструктор тестов, банк вопросов.
- **app-admin**
  - Панель управления пользователями и ролями.
  - Доступ ко всем сервисным модулям (в том числе «Сообщить об ошибке», «Заполнение справочников» и т.п.).
- **dictionary-admin**
  - Доступ к модулю заполнения справочников по транспортной документации.
  - Управление нормативно‑справочной информацией (страны, станции, грузы, плательщики, виды работ и др.).

## Основные модули фронтенда

### Главное меню

Главное меню предоставляет доступ к основным разделам тренажёра:
- **Заявка на грузоперевозку**
- **Накладная**
- **Акты**
- **Памятка приёмосдатчика**
- **Ведомости подачи и уборки**
- **Накопительная ведомость**
- **Загрузка и выгрузка документов** (единый список документов по пользователю)
- **Режим теста**
- **Сообщить об ошибке**
- **Справочник** (загрузка/поиск текстовых справочных документов)
- **Кейсы / сценарии обучения**
- **Панель преподавателя** (для `teacher` и `app-admin`)
- **Панель управления** (для `app-admin`)
- **Заполнение справочников** (для `dictionary-admin` и `app-admin`)

### Модули по работе с документацией

- **Заявка на грузоперевозку**
  - Полный набор полей учебной заявки (страны, станции, грузоотправитель, принадлежность вагонов, договоры и т.д.).
  - Интеграция со справочниками (страны, станции, юридические лица, группы грузов, способы подачи, принадлежность вагонов и др.).
  - Работа с отправками (`Sendings`) и графиком подач (SubmissionSchedule).
  - Блок **«Плательщики/Экспедиторы»**:
    - выбор страны, за перевозку по которой платят;
    - выбор плательщика/экспедитора из справочника `payers`;
    - автоматическое подхватывание ОКПО, наименования, адреса и примечания.
  - Учебный расчёт провозной платы и служебные подсказки в подвале (обязательные поля, проверки дат и т.д.).

- **Накладная**
  - Поля отправителя/получателя, виды грузовых работ, страны отправления/назначения, станции, способы оплаты, планируемый род вагона и др.
  - Привязка к справочникам грузов (ГНГ / ЕТ СНГ), отметок, документов, массы, собственников и прочей нормативной информации.

- **Акты**
  - Виды актов (общие и коммерческие).
  - Станция составления акта, отправка, номера вагонов/контейнеров, отметки отправителя о состоянии тары/груза, номер акта и другая служебная информация.

- **Памятка приёмосдатчика**
  - Учебный модуль для заполнения памятки с контролем обязательных полей и логики.

- **Ведомости подачи и уборки / Накопительная ведомость**
  - Формы ведомостей с привязкой к отправкам, станциям, плательщикам.
  - Логика проверки обязательных полей и корректности дат.

- **Загрузка и выгрузка документов**
  - Централизованный список всех документов пользователя (по типу: заявка, накладная, акты и др.).
  - Просмотр, пометка как удалённых (soft‑delete) и восстановление.
  - Экспорт отдельных документов в PDF.

### Справочники

Справочные данные используются всеми документами и доступны для просмотра/редактирования через отдельные модули.

- **Справочник** (контент‑модуль)
  - Загрузка файлов справочных материалов (PDF/Word).
  - Полнотекстовый поиск по названию и содержимому.
  - Просмотр и скачивание документов (с проверкой наличия файла).

- **Заполнение справочников по работе с транспортной документацией** (роль `dictionary-admin`/`app-admin`)
  - Вступительный экран с описанием и инструкцией.
  - **Секция «Обновление справочников»**:
    - выбор конкретного справочника (страны, станции, юридические лица, группы грузов, грузы, владельцы путей, виды упаковки, виды подвижного состава, плательщики/экспедиторы, виды отправок, признаки отправки, способы подачи, виды скорости, договоры и др.);
    - просмотр текущих записей;
    - добавление и редактирование строк (с «человеческими» подписями полей).
  - **Секция «Удаление данных»**:
    - выбор справочника;
    - безопасное удаление записей (с предупреждением о влиянии на документы).
  - **Секция «Просмотр данных»**:
    - слева список справочников;
    - справа таблица с содержимым выбранного справочника.

### Режим теста

- **Роль студента**
  - Просмотр доступных тестов.
  - Прохождение теста в интерфейсе SurveyJS.
  - Поддержка изображений в вопросах (через API, без `blob:`‑строк в тексте).
  - Свободный ввод:
    - нормализация регистра и первой буквы;
    - учёт эквивалентности `е`/`ё`;
    - специальные правила для `нн`/`сс` при проверке.
  - Расчёт баллов:
    - разболовка по вопросам;
    - вычисление процента прохождения и статуса **«Тест сдан / не сдан»**;
    - отображение процента и набранных баллов.
  - Журнал неправильных ответов:
    - таблица с вопросами, ответом студента и правильным(и) ответом(ами).

- **Роль преподавателя**
  - Банк вопросов:
    - создание и редактирование вопросов (включая вложенные изображения).
  - Конструктор тестов:
    - сборка тестов из вопросов и вариантов;
    - настройка баллов за каждый вопрос;
    - установка порога прохождения в процентах;
    - задание максимального количества попыток.
  - Панель преподавателя:
    - список студентов и их попыток по тестам;
    - лучший результат в процентах (с защитой от ошибок приведения типов);
    - просмотр статуса прохождения сценариев и кейсов.

### Кейсы / сценарии

- **Кейсы (раньше «Сценарии»)** — набор учебных сценариев работы с документами:
  - режимы обучения («Новичок», «Продвинутый» и др.);
  - отдельные экраны со сценарием, тренажёром и инструкциями;
  - отслеживание просмотра кейсов студентами.

## Модуль «Сообщить об ошибке»

- Доступен всем авторизованным пользователям (создание тикетов) и `app-admin` (обработка).
- Тикет содержит:
  - имя отправителя;
  - модуль, где обнаружена ошибка;
  - описание;
  - статус (`отправлено`, `на рассмотрении`, `работаем над ошибкой`, `ошибка исправлена`);
  - текст ошибки из DevTools (или «ошибки в DevTools нет»);
  - ответ администратора.
- **Секция «Сообщить об ошибке»**:
  - форма отправки тикета;
  - список собственных тикетов пользователя с отображением статуса и ответа.
- **Секция «Сообщения об ошибках»** (для `app-admin`):
  - таблица всех тикетов со временем создания;
  - просмотр деталей;
  - изменение статуса и добавление ответа.

## Администрирование и интеграция с Keycloak

- **Панель управления (AdminPanel)**
  - Список пользователей из Keycloak (логин, ФИО, email, роли).
  - Управление ролями:
    - назначение/снятие ролей `student`, `teacher`;
    - назначение/снятие роли `dictionary-admin`.
  - Все операции выполняются через Keycloak Admin REST API.

- **Аутентификация и авторизация**
  - Все API‑маршруты защищены через `verifyToken()`; доступ по ролям реализован через:
    - `requireRealmRole('app-admin')`;
    - `requireAnyRealmRole([...])` для комбинированного доступа.
  - На фронте используется Keycloak JS‑адаптер:
    - инициализация с `onLoad: 'check-sso'`;
    - ручной логин при отсутствии сессии;
    - периодическое обновление токена;
    - вспомогательные функции `isStudent`, `isTeacher`, `isAppAdmin`, `isDictionaryAdmin`, `hasAnyRealmRole`.

## Этапы настройки

### 1. Keycloak и БД через Docker (рекомендуется)
В каталоге `projirgups` выполните:
```bash
docker-compose up -d
```
Дождитесь запуска сервисов (30–60 секунд). Логи Keycloak:
```bash
docker-compose logs -f keycloak
```
Реалм задаётся в `.env` (переменная `KEYCLOAK_REALM`, по умолчанию `master`). Импорт/экспорт реалма — см. [KEYCLOAK_REALM_IMPORT_EXPORT.md](./KEYCLOAK_REALM_IMPORT_EXPORT.md).

### 2. Настройте Keycloak

1. Откройте консоль администратора Keycloak: `http://localhost:8080`
2. Войдите под админом:
   - Логин: `admin`
   - Пароль: `admin`
3. Создайте клиента:
   - Перейдите в **Клиенты** → **Создать клиента**
   - Идентификатор клиента: `irtran-client`
   - Протокол клиента: `openid-connect`
   - Нажмите **Далее** и настройте:
     - **Тип доступа**: `public`
     - **Стандартный поток**: `ON`
     - **Предоставление прямого доступа**: `ON`
     - **Допустимые URI перенаправления**:
       - `http://localhost:5173/*`
       - `http://localhost:3000/*`
     - **Веб-источники**: `http://localhost:5173`
   - Нажмите **Сохранить**

4. (Опционально) Настройте SMTP для писем:
   - **Настройки области (Realm Settings)** → вкладка **Электронная почта**
   - Пример для Gmail:
     - **Хост**: `smtp.gmail.com`
     - **Порт**: `587`
     - **Включить StartTLS**: `ВКЛ`
     - **Включить аутентификацию**: `ВКЛ`
     - **Имя пользователя**: ваш Gmail адрес
     - **Пароль**: пароль приложения Gmail

   Примечание для Gmail:
   - Включите 2FA в аккаунте Google
   - Сгенерируйте пароль приложения: `https://myaccount.google.com/apppasswords`

5. (Опционально) Настройте подтверждение e-mail:
   - **Настройки Realm** → вкладка **Вход**: включите **Регистрацию пользователя** (если требуется)
   - **Настройки Realm** → вкладка **Электронная почта**: включите **Подтверждение электронной почты**
   - **Аутентификация** → **Потоки** → **Регистрация**: убедитесь, что шаг подтверждения e-mail включён

### 3. Установить зависимости

cd irtran_server-main

### 5. Настройка сервера

Конфигурация находится в `irtran_server-main/config/app.json`:

```json
{
  "server_name": "localhost",
  "port": 3000,
  "use_https": false,
  "enable_auth": true
}
```

Конфигурация Keycloak находится в `irtran_server-main/config/keycloak.json`:

```json
{
  "realm": "master",
  "auth-server-url": "http://localhost:8080",
  "resource": "irtran-client"
}
```

### 6. Настройка клиента

Конфигурация клиента Keycloak находится в `IrTRAN-main/src/helpers/keycloak.js`:

```javascript
const keycloakConfig = {
  url: 'http://localhost:8080',
  realm: 'master',
  clientId: 'irtran-client'
};
```

cd irtran_server-main

Сервер: `http://localhost:3000`

### 8. Запустить клиент (фронт)

```bash
cd IrTRAN-main
npm run dev
```
Клиент: `http://localhost:5173`

## Устранение неполадок

### Keycloak не запускается
- Проверьте порты `8080` (Keycloak) и порт БД из `docker-compose.yml`
- Проверьте логи Docker: `docker-compose logs keycloak`

### Ошибки аутентификации
- Убедитесь, что Keycloak запущен: `http://localhost:8080`
- Проверить конфигурацию клиента в консоли администратора Keycloak
- Убедиться, что URI перенаправления совпадают
- Проверить консоль браузера на наличие ошибок

### Ошибки аутентификации сервера
- Убедиться, что `enable_auth` установлен в значение `true` в `config/app.json`
- Проверить конфигурацию Keycloak в `config/keycloak.json`
- Убедиться, что идентификатор клиента совпадает в конфигурациях сервера и клиента

### Электронная почта не работает
- Убедиться, что пароль приложения Gmail верен
- Проверьте журналы Keycloak на наличие ошибок SMTP
- Убедиться, что двухфакторная аутентификация (2FA) включена в учётной записи Gmail

## Отключение аутентификации (разработка)
Чтобы отключить аутентификацию для разработки:

1. Установить `enable_auth: false` в `irtran_server-main/config/app.json`
2. Сервер будет принимать запросы без токенов

## Примечания
- Сервер работает по протоколу HTTP (порт 3000) для локальной разработки
- Keycloak работает по протоколу HTTP (порт 8080) для локальной разработки
- Для работы настройте HTTPS и соответствующим образом обновите URL-адреса
- Для подтверждения адреса электронной почты требуется правильная настройка SMTP
